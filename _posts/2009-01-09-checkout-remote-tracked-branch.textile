Die folgenden Schritte führe ich regelmäßig durch, wenn ich meine zahlreichen
Computer aufsetze oder synchronisiere und jedesmal scheine ich die Kommandos
dafür zu vergessen. So laß uns annehmen, dass du mehr als einen Zweig in deinem
entfernten Repository hast und du möchtest ihn auch in dein lokales Repository
einbringen:

p=. !http://gitready.com/images/branches.png!

Das Anzeigen von Informationen in einem entfernten Repository sieht in etwa wie
folgt aus:

<pre>
$ git remote show origin
  * remote origin
    URL: *************
    Remote branch merged with 'git pull' 
      while on branch master
        master
      Tracked remote branches
        haml master
</pre>

Glücklicherweise ist die Kommandosyntax hierfür recht einfach:

<pre>
git checkout --track -b 
  <local branch> <remote>/<tracked branch>
</pre>

In meinem Fall benutze ich das Kommando:

@git checkout --track -b haml origin/haml@

Du kannst auch eine einfacherer Variante verwenden:

@git checkout -t origin/haml@
